# 🚀 Установка и запуск IThub School

Подробная инструкция по установке и запуску приложения электронного журнала.

## 📋 Требования

Перед установкой убедитесь, что у вас установлены:

- **Node.js** версии 14.0 или выше ([скачать](https://nodejs.org/))
- **npm** версии 6.0 или выше (обычно устанавливается вместе с Node.js)
- **Git** ([скачать](https://git-scm.com/))
- Любой современный браузер (Chrome, Firefox, Safari, Edge)

## 🔍 Проверка установки

Откройте терминал и выполните команды:

```bash
# Проверка Node.js
node --version
# Должно вывести версию, например: v16.13.0

# Проверка npm
npm --version
# Должно вывести версию, например: 8.1.0
```

## 📥 Установка приложения

### Вариант 1: С GitHub (если выложено на GitHub)

```bash
# Клонируйте репозиторий
git clone https://github.com/ваш-username/ithub-journal.git

# Перейдите в папку проекта
cd ithub-journal

# Установите зависимости
npm install
```

### Вариант 2: Из загруженного архива

1. Распакуйте архив в удобное место
2. Откройте терминал в папке проекта
3. Выполните команду:
```bash
npm install
```

## ▶️ Запуск приложения

### Запуск в режиме разработки

```bash
npm start
```

Приложение автоматически откроется в браузере по адресу:
```
http://localhost:3000
```

Если браузер не открылся автоматически, откройте его вручную и перейдите по адресу выше.

**Особенности режима разработки:**
- Приложение автоматически перезагружается при изменении файлов
- Доступны инструменты отладки в DevTools (F12)
- Более подробные сообщения об ошибках

### Остановка сервера

Нажмите `Ctrl+C` в терминале, где запущен `npm start`.

## 🛠️ Сборка для продакшена

Когда приложение готово к публикации, выполните команду:

```bash
npm run build
```

Это создаст оптимизированную сборку в папке `build/`:
- Минифицированный код
- Оптимизированные изображения
- Лучшая производительность
- Полная готовность к развёртыванию

## 🌐 Развёртывание

### На GitHub Pages

1. Создайте новый репозиторий на GitHub

2. Откройте файл `package.json` и добавьте:
```json
"homepage": "https://github.com/ваш-username/ithub-journal"
```

3. В терминале выполните:
```bash
npm install --save-dev gh-pages
```

4. Добавьте скрипты в `package.json` (в раздел "scripts"):
```json
"deploy": "npm run build && gh-pages -d build",
"predeploy": "npm run build"
```

5. Выполните развёртывание:
```bash
npm run deploy
```

6. В параметрах репозитория включите GitHub Pages

### На Vercel (рекомендуется)

Vercel - это платформа для развёртывания React приложений:

1. Зарегистрируйтесь на [vercel.com](https://vercel.com)

2. Нажмите "New Project"

3. Импортируйте свой GitHub репозиторий

4. Выберите эту папку проекта

5. Нажмите "Deploy"

Приложение будет развёрнуто автоматически!

### На собственном сервере

1. Соберите приложение: `npm run build`

2. Загрузите содержимое папки `build/` на сервер (FTP, SSH, и т.д.)

3. Настройте веб-сервер (Nginx, Apache):

**Nginx пример:**
```nginx
server {
    listen 80;
    server_name ваш-домен.com;
    
    root /var/www/ithub-journal/build;
    index index.html;
    
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

## 🐛 Устранение неполадок

### Ошибка: "npm: command not found"

**Решение:** Node.js и npm не установлены или не добавлены в PATH.
1. Скачайте и установите Node.js с [nodejs.org](https://nodejs.org/)
2. Перезагрузите компьютер
3. Проверьте установку: `node --version`

### Ошибка: "Port 3000 is already in use"

**Решение:** Другой процесс использует порт 3000.

Вариант 1 (Windows PowerShell):
```powershell
# Найдите процесс на порту 3000
netstat -ano | findstr :3000

# Завершите процесс (замените PID на реальный номер)
taskkill /PID <PID> /F
```

Вариант 2 (Mac/Linux):
```bash
# Завершите процесс на порту 3000
lsof -ti:3000 | xargs kill -9
```

Вариант 3 (использование другого порта):
```bash
PORT=3001 npm start
```

### Ошибка: "Cannot find module 'react'"

**Решение:** Зависимости не установлены.

```bash
# Удалите папку node_modules
rm -rf node_modules

# Переустановите зависимости
npm install
```

### Белый экран после входа

**Решение:**
1. Откройте DevTools (F12)
2. Перейдите на вкладку Console
3. Проверьте наличие ошибок
4. Убедитесь, что все CSS файлы загружены
5. Попробуйте очистить кэш браузера: `Ctrl+Shift+Delete`

### Стили не применяются

**Решение:**
1. Убедитесь, что все CSS файлы в папке `src/pages/` и `src/components/`
2. Проверьте пути импорта в файлах JSX
3. Перезагрузите страницу: `Ctrl+F5` (полная перезагрузка)
4. Очистите кэш браузера
