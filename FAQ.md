# ❓ FAQ - Часто задаваемые вопросы

Ответы на частые вопросы о IThub School React приложении.

---

## 🚀 Установка и запуск

### В: Как запустить приложение?
**О:** 
1. Откройте терминал в папке проекта
2. Выполните команды:
```bash
npm install
npm start
```
3. Приложение откроется на http://localhost:3000

### В: Что означает ошибка "npm: command not found"?
**О:** Node.js не установлен. Установите его с https://nodejs.org/

Проверьте установку:
```bash
node --version
npm --version
```

### В: Как закрыть приложение?
**О:** В терминале нажмите `Ctrl+C` (или `Cmd+C` на Mac)

### В: Почему приложение медленно запускается первый раз?
**О:** Node.js скачивает и устанавливает все зависимости. Это нормально. При следующих запусках быстрее.

## 🔐 Вход и аккаунты

### В: Какой пароль используется?
**О:** Пароли НЕ требуются! Просто введите email и выберите роль. Это тестовое приложение.

### В: Есть ли правда какие-то демо аккаунты?
**О:** Да! Используйте любые из этих:
- **Студент:** alexey@ithub.school
- **Учитель:** lyudmila@ithub.school
- **Родитель:** parent@ithub.school
- **Админ:** admin@ithub.school

### В: Почему я не могу войти с другим email?
**О:** В тестовом приложении работают только 4 демо-аккаунта выше.

### В: Как выйти из аккаунта?
**О:** Нажмите кнопку "Выход" в навигации (правый верхний угол).

### В: Сохранится ли мой вход при перезагрузке страницы?
**О:** Да! Данные хранятся в localStorage браузера и восстановятся при перезагрузке.

## 📊 Функциональность страниц

### Страница "Оценки"

**В: Почему я не могу добавить оценку?**
**О:** Только учителя могут добавлять оценки. Залогинитесь как `lyudmila@ithub.school`

**В: Как удалить оценку?**
**О:** Нажмите кнопку "Удалить" (иконка корзины) в строке с оценкой

**В: Как работает поиск?**
**О:** Вводите название предмета или имя учителя в поле поиска. Поиск работает в реальном времени.

**В: Почему некоторые оценки другого цвета?**
**О:** Цвета зависят от значения оценки:
- Зелёный = 5 (отлично)
- Синий = 4 (хорошо)
- Жёлтый = 3 (удовлетворительно)
- Красный = 2 (неудовлетворительно)

### Страница "Расписание"

**В: Как переключаться между днями недели?**
**О:** Нажимайте на кнопки дней (Пн, Вт, Ср, Чт, Пт)

**В: Почему некоторые уроки серые?**
**О:** Это перерывы между уроками. На них нет предмета и учителя.

**В: Как узнать время уроков?**
**О:** Посмотрите таблицу "График звонков" внизу страницы

### Страница "Домашние задания"

**В: Как отметить задание выполненным?**
**О:** Нажмите кнопку "Выполнено" на карточке задания

**В: Что значит статус "Просрочено"?**
**О:** Deadline прошёл, а задание ещё не выполнено

**В: Как удалить задание?**
**О:** Нажмите кнопку "Удалить" (иконка корзины)

**В: Почему указано "дни осталось"?**
**О:** Это количество дней до deadline. Если отрицательное - уже просрочено.

### Страница "Профиль"

**В: Как отредактировать профиль?**
**О:** Нажмите кнопку "Редактировать" и измените поля

**В: Почему я не могу изменить email?**
**О:** Email - первичный идентификатор аккаунта, поэтому не редактируется

**В: Как сохранить изменения в профиле?**
**О:** Нажмите кнопку "Сохранить" (она появляется в режиме редактирования)

### Админ панель

**В: Почему я не вижу админ панель?**
**О:** Админ панель видна только для администраторов. Залогинитесь как `admin@ithub.school`

**В: Как удалить пользователя?**
**О:** На вкладке "Пользователи" нажмите кнопку "Удалить" в строке с пользователем

**В: Как добавить нового пользователя?**
**О:** Нажмите кнопку "Добавить пользователя" на вкладке "Пользователи"

## 💾 Данные и сохранение

### В: Где хранятся мои данные?
**О:** В localStorage браузера. Это локальное хранилище, не облако.

### В: Если я очищу кэш браузера, потеряются ли данные?
**О:** Да! Обычно это происходит при "Очистить историю браузера"

### В: Как сделать резервную копию данных?
**О:** Данные в tестовом приложении условные. В реальном приложении будет сохранение на сервере.

### В: Может ли кто-то получить доступ к моим данным?
**О:** Только с этого компьютера, т.к. данные в localStorage. При удалении браузера/cookies они теряются.

## 🐛 Ошибки и проблемы

### В: Я вижу красную ошибку в консоли браузера (F12)
**О:** 
1. Обновите страницу (F5)
2. Закройте приложение (Ctrl+C) и запустите снова (`npm start`)
3. Очистите кэш браузера (Ctrl+Shift+Delete)

### В: Приложение не запускается. Что делать?
**О:** Проверьте ошибки в терминале:
1. Убедитесь, что Node.js установлен: `node --version`
2. Удалите папку `node_modules` и `package-lock.json`
3. Переустановите: `npm install`
4. Запустите: `npm start`

### В: Ошибка "EACCES: permission denied"
**О:** На Mac/Linux могут быть проблемы с правами. Попробуйте:
```bash
sudo npm install
# или
npm install --verbose
```

### В: Порт 3000 занят, что делать?
**О:** Закройте приложение, которое использует порт 3000, или запустите на другом порту:
```bash
PORT=3001 npm start
```

### В: Стили не загружаются / всё белое
**О:** 
1. Очистите кэш браузера (Ctrl+Shift+Delete)
2. Нажмите Ctrl+F5 (полная перезагрузка)
3. Закройте приложение и запустите снова

### В: Таблица на мобильном выглядит ужасно
**О:** Это нормально при очень узком экране. Разработчик может улучшить мобильный дизайн позже.

## 📦 Развёртывание и GitHub

### В: Как выложить приложение на GitHub?
**О:** Следуйте инструкциям в файле `DEPLOY.md`:
1. Создайте репозиторий на GitHub
2. Инициализируйте Git локально
3. Загрузите код на GitHub
4. Разверните на GitHub Pages или Vercel

### В: Что такое GitHub Pages?
**О:** Это бесплатная служба GitHub для хостинга статических сайтов. Идеально для React приложений.

### В: Что такое Vercel?
**О:** Это платформа для развёртывания React приложений. Очень простая в использовании и бесплатная.

### В: Какой способ развёртывания выбрать?
**О:** 
- **GitHub Pages** - если хотите всё на GitHub
- **Vercel** - если хотите максимум удобства
- Оба работают отлично!

### В: После развёртывания приложение показывает 404
**О:** 
1. Проверьте поле `"homepage"` в `package.json`
2. Убедитесь, что ветка `gh-pages` (для GitHub Pages) или `main` (для Vercel) указана правильно
3. Очистите кэш браузера

### В: Как обновить приложение на GitHub/Vercel?
**О:** 
```bash
git add .
git commit -m "Описание изменений"
git push
```
Приложение обновится автоматически!

## 📚 Разработка и кастомизация

### В: Как добавить новую страницу?
**О:** 
1. Создайте файл `src/pages/NewPage.jsx` с компонентом
2. Создайте файл `src/pages/NewPage.css` для стилей
3. Импортируйте в `src/App.jsx`
4. Добавьте логику переключения на новую страницу

### В: Как изменить цветовую схему?
**О:** Отредактируйте `:root` переменные в `src/App.css`:
```css
:root {
  --primary-color: #9D7BD1;
  --secondary-color: #7B5CA8;
  /* и т.д. */
}
```

### В: Как добавить новый демо-аккаунт?
**О:** В `src/App.jsx` найдите объект `demo_accounts` и добавьте новый:
```javascript
{
  email: 'new@ithub.school',
  role: 'student'
}
```

### В: Как интегрировать реальную БД?
**О:** 
1. Установите Redux или Context API для состояния
2. Интегрируйте API (fetch или axios)
3. Замените локальные данные на API запросы

Это выходит за рамки этого FAQ - обратитесь к документации разработчика.

### В: Как добавить аутентификацию с пароля?
**О:** Это выходит за рамки тестового приложения. Для продакшена нужен реальный backend с JWT токенами.
