# Diploma Project: Student Personal Records Management System

## 📚 Описание проекта

Веб-интерфейс для управления личными делами учащихся в школьной образовательной организации. Система позволяет администраторам и учителям вести учёт личных дел, отслеживать проблемы и достижения учащихся, управлять документами.

## ✨ Ключевые особенности

- ✅ **Управление учащимися** - добавление, редактирование, удаление, поиск
- ✅ **Личные дела** - создание, редактирование, категоризация
- ✅ **Приоритизация** - низкий, средний, высокий приоритет
- ✅ **Статусы** - открыто, закрыто, архив
- ✅ **Аутентификация** - безопасный вход для администраторов и учителей
- ✅ **История изменений** - аудит всех операций
- ✅ **ELLE-inspired дизайн** - элегантный, женственный, премиальный интерфейс в стиле журнала ELLE
- ✅ **Адаптивный дизайн** - идеально работает на всех устройствах
- ✅ **Темная/светлая тема** - комфортное использование в любое время суток
- ✅ **Плавные анимации** - современные переходы и микро-взаимодействия
- ✅ **Полнофункциональный** - всё, что нужно для работы

## 🚀 Быстрый старт

### 1. Установка

```bash
# Клонируем и переходим в папку
cd DIPLOM

# Устанавливаем все зависимости
npm install
cd frontend && npm install && cd ..
```

### 2. Запуск

**Вариант 1 - Оба сервера одновременно:**
```bash
npm run dev
```

**Вариант 2 - Отдельно:**
```bash
# Терминал 1
npm start

# Терминал 2
npm run client
```

### 3. Открыть приложение

```
http://localhost:3000
```

### 4. Вход

- **Пользователь:** admin
- **Пароль:** admin123

## 🔄 API Endpoints

### Authentication
- `POST /api/auth/login` - Вход в систему
- `GET /api/auth/me` - Получить текущего пользователя

### Students
- `GET /api/students` - Список учащихся
- `POST /api/students` - Создать учащегося
- `GET /api/students/:id` - Получить учащегося
- `PUT /api/students/:id` - Обновить учащегося
- `DELETE /api/students/:id` - Удалить учащегося

### Records
- `GET /api/records/student/:studentId` - Личные дела учащегося
- `POST /api/records` - Создать личное дело
- `GET /api/records/:id` - Получить личное дело
- `PUT /api/records/:id` - Обновить личное дело
- `DELETE /api/records/:id` - Удалить личное дело

### Documents
- `GET /api/documents/record/:recordId` - Документы личного дела
- `POST /api/documents` - Создать документ
- `DELETE /api/documents/:id` - Удалить документ

## 💾 База данных

SQLite3 база данных содержит следующие таблицы:

- **users** - Администраторы и учителя
- **students** - Информация об учащихся
- **personal_records** - Личные дела
- **documents** - Документы и прикрепления
- **audit_log** - История всех операций
